# Удаление chroma.sqlite3 из репозитория

Файл `chroma_db/chroma.sqlite3` весит 147.7 МБ и **не должен** храниться в репозитории, так как:
- Это временная база данных
- Она создается автоматически при первом запуске
- Ее можно пересоздать из `Template.xml`

## Если файл уже в репозитории:

### Шаг 1: Удалите файл из Git (но оставьте локально)

```bash
git rm --cached chroma_db/chroma.sqlite3
```

Или если файл в корне:
```bash
git rm --cached chroma.sqlite3
```

### Шаг 2: Убедитесь, что файл в .gitignore

Проверьте, что в `.gitignore` есть:
```
chroma_db/
*.sqlite3
*.sqlite
*.db
```

### Шаг 3: Закоммитьте изменения

```bash
git add .gitignore
git commit -m "Remove chroma.sqlite3 from repository - will be auto-generated"
git push
```

### Шаг 4: Проверьте, что файл удален из репозитория

Файл должен остаться на вашем компьютере, но больше не будет отслеживаться Git.

## При деплое:

При развертывании приложения база данных `chroma.sqlite3` будет создана автоматически при первом запуске, если вы используете функцию создания векторной БД.

---

**Примечание:** Если вы используете Git LFS и хотите хранить базу данных (не рекомендуется), см. `GIT_LFS_SETUP.md`.
